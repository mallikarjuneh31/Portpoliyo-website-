# Mallikarjun's Portfolio

A modern, responsive portfolio website built with React that showcases my skills, projects, and experience as a frontend developer and computer science engineering student.

## Features

- **Modern UI/UX Design**: Clean, professional design with attention to detail
- **Responsive Layout**: Works seamlessly on all device sizes
- **Interactive Elements**: Smooth animations and hover effects
- **Dark/Light Theme**: Toggle between color schemes with persistent preference
- **Visitor Counter**: Tracks unique visitors to the portfolio
- **Project Showcase**: Dedicated section for featured projects
- **Skills Display**: Organized presentation of technical abilities
- **Education Timeline**: Chronological display of academic achievements
- **Contact Form**: Easy way for visitors to get in touch

## Technologies Used

- **React**: Component-based UI library
- **CSS3**: Advanced styling with Flexbox and Grid layouts
- **JavaScript**: Interactive functionality
- **Font Awesome**: Icon library for visual elements
- **Google Fonts**: Custom typography

## Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/Mallikarjun-9610/portfolio.git
   ```

2. Navigate to the project directory:
   ```bash
   cd portfolio
   ```

3. Install dependencies:
   ```bash
   npm install
   ```

4. Start the development server:
   ```bash
   npm start
   ```

5. Open [http://localhost:3000](http://localhost:3000) to view the portfolio in your browser.

## Customization

You can easily customize this portfolio by modifying:

- **Personal Information**: Update the content in `src/App.js`
- **Colors**: Modify the CSS variables in `src/App.css`
- **Projects**: Add or modify project entries in the Projects section
- **Skills**: Update the skills list in the Skills section
- **Contact Information**: Change contact details in the Contact section

## Deployment

To deploy this portfolio to GitHub Pages or another hosting service:

1. Build the project:
   ```bash
   npm run build
   ```

2. The optimized build will be in the `build` folder, ready for deployment.
